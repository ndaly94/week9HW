1. 1. Explain what a REST API is. What makes an API "RESTful"?
  2. REST stands for Representational State Transfer. It is an architecture style for web-based APIs. Also known as a runtime environment. It works by storing representations of a code on the server that the browser can access via HTTP methods.
  3. 2. What is Node.js and why is it often used when building server-side applications?
  4. Node lets us run JavaScript outside of the browser environment. It is lightweight module and doesn’t have a DOM or web APIs. It is updated and expanded upon via packages that are shared via open-source devs online.
  5. 3. Express.js is a popular framework used in conjunction with Node.js. What are some of the key features of Express.js that make it useful for building web servers?
  6. Express is minimalistic and lightweight compared to other popular frameworks. It also works with Node’s built-in HTTP module making it seamless integration for Node-based apps.
  7. 4. In Express.js, what is middleware and how is it used?
  8. Middleware is programs that facilitate communication and interaction between applications and the system software or between a client and server. In our express example, we use its JSON parser which allows our machine to read JSON files accurately.
  9. 5. Regarding HTTP verbs, what is the difference between GET, POST, PUT, PATCH, and DELETE? When would you use each in the context of a RESTful API?
  10. While they all have different purposes, GET retrieves data from the server and renders it in the browser. POST allows us to input data in the browser and have it stored in the server based on the rules the route set up for us. PUT is similar to POST except it works for updating already stored info. PUT requires a full update of all the info while PATCH allows us to do a partial update. And DELETE do what you think it do.
